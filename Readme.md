# Шаблоны проектирования
***Design Patterns***

**Шаблон проектирования - это именованное эффективное решение характерной задачи проектирования программного обеспечения.**

*Зачастую, задачи, которые необходимо решать в программировании, повторяются. Использование шаблонов позволяет:* 
> - не изобретать велосипед и писать "чистый" код;
> - упростить будущий рефакторинг кода, при необходимости;
> - использовать одинаковую терминологию при обсуждении софта;
> - обеспечить понимание кода другими программистами.

---
*В данном проекте обзорные примеры 23-х шаблонов, которые подробно описывались в книге "Design Patterns. Elements of Reusable Object-Oriented Software" Банды Четырёх.*

| № | Название шаблона | Для чего используется|Описание примера|
|:----:|:----:|:---------|:---------|
|  1 | Template method (Шаблонный метод) | Используется для вычленения и локализации общего для всех подклассов поведения в родительском. Подклассы же переопределяют некоторые части родительского не изменяя его структуру в целом | В примере рабочий день двух разработчиков. Общая часть этого дня вынесена в родительский класс, а отличная - переопределяется наследниками |
|  2 | Prototype (Прототип) | Используется для созданий копий объектов | В примере мы клонируем человека |
|  3 | FactoryMethod (Фабричный метод) | Используется для делегирования создания экземпляров другому классу | В примере по модели автомобиля создаётся объект победителя в гонках с помощью другого класса |
|  4 | State (Состояние) | Используется для того, чтобы управлять поведением объекта в зависимости от его внутреннего состояния | В примере женщина, поведение которой меняется в зависимости от её настроения |
|  5 | Observer (Наблюдатель) | Используется для определения зависимости «Один ко многим» между объектами таким образом, что при изменении состояния одного объекта все зависящие от него объекты были увндомлены и обновились | В примере Youtube канал с подписчиками (наблюдателями). При появлении нового видео на калане, подписчикам приходит уведомление |
|  6 | Command (Команда) | Используется, чтобы задать параметры клиентов для обработки определённых запросов, создания очереди из этих запросов или их консторя и поддержки отмены операций | В примере игрушечная машинка, которая может ехать вперёд, назад, вправо и влево. И создаётся пульт дистанционного управления это машинкой |
|  7 | AbstractFactory (Абстрактная фабрика) | Используется для создания множеств взаимосвязанных объектов без привязки к конкретным классам | В примере с помощью TeamFactory создаётся 2 команды спорткаров и миниавто, которые принимают участие в своих гонках |
|  8 | Singleton (Одиночка) | Используется для создания единственного экземпляра конкретного класса | В примере создан класс, который ведёт датированную хронологию по ежедневному обучениию. А также тотальный итог в часах за весь период обучения |
|  9 | Builder (Строитель) | Используется для отделения конструирования сложного объекта от его представления. В результате из одного и того же конструирования мы можем получить разные представления | В примере создаются разные ргуппы в дестком саду из одного шаблона-строителя |
| 10 | Adapter (Адаптер) | Используется для обеспечения совместной работы классов, интерфейсы которых не совместимы | В примере База Данных принимает методы, которые находятся в консольном CRUD приложении |
| 11 | Bridge (Мост) | Используется для получения преимуществ наследования без потери гибкости | В примере мы отправляем бригады рабочих в командировки в разные страны. Имеем возможность добавления новых рабочих и новые командировки |
| 12 | Composite (Компоновщик) | Используется для компоновки медких объектов в более крупные, которые в свою очередь могут стать основой для ещё более крупных структур (по типу «дерева») | В примере мы создали 2 команды по 4 спортсмена, которые участвуют в спортивной эстафете |
| 13 | Decorator (Декоратор) | Используется: 1) в качестве альтернативы порождению подклассов для расширения функциональности. Как следствие получаем возможность для динамического изменения обязанностей объектов; 2) если расширение класса путём создания подклассов невозможно | В примере реализована иерархия работников. Каждый из которых может выполнять свою работу и работу нижестоящих подчинённых |
| 14 | Facade (Фасад) | Используется для упрощения использования подсистемы. Изолируя клиентов от компонентов подсистемы, мы упрощаем работу с ней | В примере используерся отредактированный код из Компоновщика. Создана отдельная сущность (фасад), которая разгрузила класс-клиент |
| 15 | Flyweight (Приспособленец) | Используется для поддержки множества мелких объектов в случае, когда приложение не зависит от идентичности объекта | В примере кадровое агенство поставляет заказчику нужное количество работников заданных специальностей |
| 16 | Proxy (Прокси) | Используется для обеспечения контроля доступа к объекту в случаях, когда это необходимо. Например, локально представлять объект в другом адресном пространстве или заменить обычный указатель | В примере мы создаём Прокси для обеспечения загрузки и установки приложения по указанной ссылке ТОЛЬКО после запуска соответствующего метода |
| 17 | Chain of Responsibility (Цепочка ответственности) | Исполузуется для избежания привязки отправителя запроса к его получателю. Это даёт возможность обработать запрос нескольким объектам | В примере реактор высокого давления. В зависимости от изменения в нём рабочего давления срабатывает соотсетствующая сигнализация |
| 18 | Interpreter (Интерпретатор) | Используется для определения представления грамматики заданного языка и интерпретации его предложений | В примере из условно большого количесва статей мы вначале отсеиваим те, что о языке java, затем из них ищем статьи конкретно о методе walkFileTree. Делаем это по ключевым словам |
| 19 | Iterator (Итератор) | Используется для последовательного доступа ко всем элементам составного объекта, скрывая его внутреннее представление | В примере мы выводим в консоль информацию о Республике Беларусь и итерируемся по её областям |
| 20 | Mediator (Посредник) | Используется для определения объекта, который инкапсулирует способ взаимодействия множества объектов и обеспечения слабой связи между этими объектами | В примере реализован способ взаимодействия меджу складом одежды (он же посредник), интернет-магазинами, которые продают эту одежду и поставщиками |
| 21 | Memento (Хранитель) | Используется для фиксирования внутреннего состояния объекта за его пределами не нарушая инкапсуляцию и восстановления объекта в случае необходимости | В примете условный текстовый документ Word, который в любое время мы можем сохранить и восстановить сохранённыю версию. А также можем использовать режим автосохранения с указанием интервала между сохранениями |
| 22 | Strategy (Стратегия) | Используется, когда есть родственные классы, которые отличаются поведением. И необходимо организовать их взаимозаменияемость | В примере собака, активность которой устанавливается и выполняется в клиентском коде |
| 23 | Visitor (Посетитель) | Используется для выполнения разных операций с объектами многих классов одной структуры. Эти операции могут быть не связаны между собой или могут зависеть от конкретных классов. При этом можно добавлять новые операции без изменения классов структуры | В примере триатлон, три этапа которого выполняются спортсменами из разных стран и по разному. Количество этапов, спортсменов и способа выполнения в данном проекте легко изменяемы и дополняемы |
